devflow-cli 1.4.0 is here with meaningful improvements to how you set up, configure, and use the tool day to day. This release focuses on giving you more control over your workflow preferences, a cleaner experience when reviewing your changes before committing, and better reliability across the board.

**What's new in 1.4.0:**

- **Choose your AI provider** — You can now select from multiple AI backends when using devflow-cli, giving you the flexibility to work with the provider that best fits your team's setup or preferences.

- **Smarter setup experience** — The initial setup process now guides you through selecting your preferred language and commit message style, so the tool speaks your team's language from day one.

- **Cleaner commit review screen** — When you're about to commit your work, files are now displayed in organized groups with short descriptions, making it much easier to understand what's changing at a glance before you confirm.

- **Automatic update notifications** — devflow-cli will now let you know when a newer version is available as soon as you open it, so you never miss an improvement.

**Under the hood improvements:**

- Renamed files are now correctly recognized and handled, so your git history stays accurate when you reorganize your project.
- Error messages throughout the tool have been made clearer and more actionable.
- Template loading has been fixed to work reliably regardless of how or where the tool is installed.
- Command execution is now more stable and consistent across different operating systems and environments.

As always, updating to the latest version is recommended to get the best experience. Thank you for using devflow-cli — your feedback continues to shape how the tool evolves.
