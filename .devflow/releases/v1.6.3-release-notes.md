devflow-cli version 1.6.3 is a focused reliability update that makes the commit workflow smarter and quieter. This release addresses two friction points that could cause unexpected behavior when organizing and saving your work.

Highlights:

- Smarter commit grouping: When you use a plan to organize your commits, devflow-cli now cross-checks your plan against the files you've actually staged. Any staged files that weren't covered by your plan are automatically included in the last commit group, so nothing gets left behind. If none of your plan groups match what's staged, the tool gracefully falls back to creating a single commit rather than failing silently or producing an empty result.

- Cleaner, quieter output: A stray warning message ("no stdin data") that would occasionally appear during normal use has been suppressed. The tool now handles background input more gracefully, keeping your terminal output clean and focused on what matters.

This release requires no changes to how you use devflow-cli — everything works the same way, just more reliably. As always, we appreciate your feedback as we continue to refine the experience.
