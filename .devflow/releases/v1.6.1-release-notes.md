devflow-cli 1.6.1 is a focused reliability release that addresses an issue affecting users who work with large repositories or run commands that return a lot of information. This update ensures that results are always shown completely, giving you the full picture every time.

- **Complete results, every time:** Previously, certain git operations could return incomplete or cut-off output when working with larger repositories or long histories. You may have noticed missing entries, truncated logs, or information that appeared to stop abruptly — that issue is now resolved.
- **Better experience at scale:** Whether you're reviewing an extensive commit history, comparing large branches, or inspecting a busy repository, you can now trust that the output you see reflects everything — nothing gets silently left behind.

This fix is particularly helpful for teams working on mature, high-activity projects where commands regularly produce a high volume of results. We recommend updating as soon as possible if you've encountered any situations where output seemed unexpectedly short or incomplete.
