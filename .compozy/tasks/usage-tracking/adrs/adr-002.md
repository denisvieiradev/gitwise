# ADR-002: PostHog Cloud free tier as the telemetry destination

## Status

Accepted

## Date

2026-07-04

## Context

The feature needs a place to store and analyze anonymous usage events. The user asked specifically for a free option. Candidate destinations were evaluated for cost at gitwise's scale, operational burden, privacy posture, and fit for anonymous CLI events. Firebase Analytics was raised initially but is a web/mobile SDK and an awkward fit for a Node CLI.

Expected volume: with opt-in telemetry on a young CLI, event volume is likely in the thousands per month.

## Decision

Use **PostHog Cloud free tier** as the telemetry destination. Events are sent as anonymous (no person profiles). The free tier (~1M events/month, no expiry) far exceeds expected volume, requires no infrastructure, and offers an EU-residency option. The event schema is kept portable so self-hosting or migration remains possible later without changing instrumentation.

## Alternatives Considered

### Alternative 1: Self-hosted PostHog

- **Description**: Run PostHog on own infrastructure via Docker.
- **Pros**: Full data ownership and residency control; strongest privacy story.
- **Cons**: Ongoing server and scaling operations; cost in ops time.
- **Why rejected**: Operational burden is unjustified at this scale; the schema stays portable so this remains a future option.

### Alternative 2: Custom lightweight endpoint

- **Description**: Build a minimal ingest endpoint plus storage and dashboards.
- **Pros**: Maximum control; minimal data by construction.
- **Cons**: Must build and maintain collection, storage, and analytics.
- **Why rejected**: Reinvents solved infrastructure with no offsetting benefit at this scale.

### Alternative 3: Firebase Analytics

- **Description**: Google's app analytics SDK.
- **Pros**: Popular, free.
- **Cons**: Built for web/mobile; awkward and heavy in a Node CLI context.
- **Why rejected**: Poor fit for the runtime and event model.

## Consequences

### Positive

- Zero cost and zero infrastructure at current and foreseeable volume.
- Purpose-built for anonymous events; fast to ship.

### Negative

- Data resides with a third-party SaaS (mitigated by EU-residency option and anonymous-only events).

### Risks

- **Outgrowing the free tier.** Mitigation: volume is far below the cap; portable schema allows migration or self-hosting if needed.

## Implementation Notes

Send events with person profiles disabled so they land in the anonymous, lower-cost tier. Generate a random install ID on first run, persisted in the gitwise config directory. Ensure events flush before process exit without blocking the user-visible command.
