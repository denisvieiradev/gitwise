# ADR-003: Big-bang rollout across all surfaces with a shared consent flag

## Status

Accepted

## Date

2026-07-04

## Context

gitwise ships on three surfaces: the `gw` CLI, the Claude Code plugin skills, and the Gemini skill mirrors. The team wants adoption and error signal from all of them. Because consent is opt-in (ADR-001) and only the CLI has a natural first-run prompt, the plugin and Gemini contexts need a consent mechanism that does not depend on prompting inside those hosts.

Rollout sequencing options were: phased by surface (CLI first), big-bang (all at once), or phased by data type (usage events first, error collection later).

## Decision

Ship telemetry across **all three surfaces in a single release**, gated behind **one shared consent flag** in `~/.gitwise/config.json`. Every surface reads the same flag and stays silent until the user has enabled telemetry (via the CLI first-run prompt, `gw config telemetry on`, or `GITWISE_TELEMETRY=1`). A user who only ever uses the plugin or Gemini surface and never enables telemetry is never tracked — the safe default.

## Alternatives Considered

### Alternative 1: Phased by surface (CLI first, then plugin + Gemini)

- **Description**: Prove consent and scrubbing on the CLI, then extend.
- **Pros**: De-risks the tricky parts on the simplest surface first.
- **Cons**: Full coverage arrives across two releases.
- **Why rejected**: The team prefers a single coordinated launch and one privacy-docs update.

### Alternative 2: Phased by data type (usage events first, error collection later)

- **Description**: Ship command-usage events first; add sanitized error/stack-trace collection later.
- **Pros**: Defers the PII-risky scrubber until proven.
- **Cons**: Delays the "where users hit errors" goal — half the objective.
- **Why rejected**: Both goals are wanted at launch.

## Consequences

### Positive

- One announcement, one privacy-docs update, complete adoption picture from day one.

### Negative

- Larger blast radius: a consent or scrubber defect ships to every surface simultaneously.

### Risks

- **A scrubber bug leaks data everywhere at once.** Mitigation: thorough scrubber test coverage and a documented, immediate off-switch before launch.
- **Plugin/Gemini users never see a prompt and so never enroll.** Accepted as the safe default; enrollment there depends on the user having enabled telemetry via the CLI or config.

## Implementation Notes

Single shared consent + install-ID + event-emitter module in `@denisvieiradev/gitwise-core`, consumed by the CLI commands, the plugin backing scripts, and the Gemini mirrors so behavior is identical across surfaces.
